# Kubernetes Notes

These manifests are starting points. Adjust namespace, image names, service
accounts, ingress, resource requests, and platform-specific policies before
production.

The execution Job spec generated by the service already sets:

- no service account token mount for execution Pods
- non-root user
- read-only root filesystem
- dropped capabilities
- `RuntimeDefault` seccomp
- ephemeral `emptyDir` mounts only
- optional `runtimeClassName`
- deny-all egress `NetworkPolicy` per run when the cluster supports it

If your cluster supports gVisor/Kata, set:

```yaml
SANDBOX_RUNTIME_CLASS_NAME: gvisor
```

The API service itself needs RBAC permissions to create/read/delete Jobs,
ConfigMaps, Pods/logs, and NetworkPolicies in the execution namespace.
